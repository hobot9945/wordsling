# Контекст проекта: Wordsling

**Суть:** Система Speech-to-Text на базе Gboard.

**Архитектура:**
1. **Клиент (Android, Kotlin):** Открывает текстовое окно с клавиатурой Gboard. Вводимый/распознанный текст сериализуется 
  и отправляется по TCP-сокету.
2. **Сервер (Rust) [ТЕКУЩИЙ ФОКУС]:** Читает данные из TCP-сокета, обрабатывает их и направляет (эмулирует ввод) в 
  сфокусированное окно на хосте.

## Протокол обмена (Wordsling Protocol)

**Логика работы:**
Клиент (Android) непрерывно транслирует состояние ввода Gboard. Передача идет "на лету", включая сырой текст, 
корректировки (возврат каретки) и фиксации (commit).

**Специальные маркеры (inline):**
1. **`[N]` (Backspace):** Команда удаления `N` символов. Генерируется, когда Gboard меняет распознанное слово "задним числом".
   *Пример:* `ехал грек чер` + `[4]а через реку` = `ехал грека через реку`.
2. **`*` (Commit):** Признак фиксации. Gboard определил, что распознавание фрагмента завершено (например, после паузы). 
  Этот кусок текста больше корректироваться не будет.

**Обычный текст:**
Всё, что не является маркерами `[N]` или `*`, передается "как есть" в кодировке UTF-8, включая пробелы и пунктуацию.

**Транспортный уровень (TCP):**
Данные передаются непрерывным потоком байт (raw stream) без разделителей кадров. Сервер читает поток по мере поступления, 
самостоятельно обрабатывая возможные разрывы многобайтовых UTF-8 символов на границах TCP-пакетов.
